## 第四部分：图片优化技术方案

### 4.1 格式选择

#### 推荐使用WEBP格式
```
对比表：
格式      文件大小  浏览器支持  质量  推荐用途
─────────────────────────────────────────────
JPG       100%      100%        中   原始/备用
PNG       300-400%  100%        高   icon/透明
WEBP      60-70%    95%+        高   ✓ 首选
AVIF      50-60%    85%         极高  未来备用

优化建议：
✓ 所有产品/场景照 → WEBP
✓ Icon/Logo → SVG (矢量优先) / PNG (备用)
✓ 备用格式：JPG (不支持WEBP的浏览器)
✓ Picture标签 fallback:
  <picture>
    <source type="image/webp" srcset="image.webp">
    <source type="image/jpeg" srcset="image.jpg">
    <img src="image.jpg" alt="...">
  </picture>
```

#### 压缩设置
```
工具：ImageOptim / TinyPNG / Squoosh
WEBP编码参数：
- 质量 (quality): 80-85 (产品图) / 75-80 (场景图)
- 方法 (method): 6 (最佳压缩，可接受的编码时间)
- 无损 (lossless): 关闭（使用有损压缩）

最终文件大小目标：
- 产品单品照 (280×280px): 15-25KB WEBP
- 产品详情主图 (600×600px): 50-80KB WEBP
- 场景照 (2560×1728px): 300-500KB WEBP
- Icon (SVG): <5KB
```

### 4.2 响应式图片策略

#### 使用Srcset和Sizes
```html
<!-- 产品卡片图片示例 -->
<img 
  src="product-640w.webp"
  srcset="product-320w.webp 320w,
          product-480w.webp 480w,
          product-640w.webp 640w,
          product-960w.webp 960w"
  sizes="(max-width: 480px) 100vw,
         (max-width: 768px) 50vw,
         (max-width: 1024px) 33.33vw,
         280px"
  alt="Luna Pendant Light"
  loading="lazy"
  decoding="async">
```

#### 图片加载策略
```
优先级1 (Eager Load) - 首屏必需：
- Hero 英雄区背景图
- 导航Logo
- 首屏产品卡片 (前4张)

优先级2 (Lazy Load) - 滚动时加载：
- 下方产品卡片
- Featured Products 以下区域
- 相关产品
- 评价头像
- 背景装饰图

优先级3 (Async Load) - 后台加载：
- Lightbox放大图
- Collections详情图
- 页脚内容

实现方案：
<img ... loading="lazy" decoding="async">
或使用 Intersection Observer API

预期效果：
- 首屏加载时间 < 2s (Largest Contentful Paint)
- 完整页面加载 < 5s
```

### 4.3 CDN和缓存策略

#### 缓存设置
```
浏览器缓存：
- 产品图：30天缓存
- 场景图：30天缓存
- Icon/Logo：1年缓存
- HTML页面：1小时缓存（或每次检查）

CDN边缘缓存：
- 所有资源：7天TTL
- 可根据用户地区自动选择最近服务器

版本控制：
文件名添加hash，如 product-abc123.webp
更新版本自动更新hash，强制客户端更新
```

### 4.4 性能指标目标

```
Core Web Vitals (Google指标):
- LCP (Largest Contentful Paint): < 2.5s
- FID (First Input Delay): < 100ms
- CLS (Cumulative Layout Shift): < 0.1

页面加载优化：
- 总页面大小：< 3MB (含所有资源)
- 图片占比：60-70% 的总大小
- 首屏加载：< 2s
- 完整加载：< 5s

成功指标：
✓ 桌面版 Lighthouse 评分：90+
✓ 移动版 Lighthouse 评分：80+
✓ 首页加载时间：1.5-2.5s (平均网络)
```

## 第五部分：图片管理工作流

### 上传和管理流程
```
1. 原始图片采集
   ↓
2. 初步筛选和分类 (A/B/C/D/E)
   ↓
3. 色彩校正和修图
   ├─ 白底处理（去背或添加背景）
   ├─ 色彩校准（确保色温一致）
   ├─ 对比度和亮度调整
   └─ 去除水印或不需要的元素
   ↓
4. 裁剪和缩放
   ├─ 根据用途裁剪（1:1, 16:9, 4:3等）
   └─ 创建多分辨率版本
   ↓
5. 格式转换和压缩
   ├─ JPG → WEBP (主格式)
   ├─ 质量设置：80-85%
   └─ 生成多分辨率：480w, 768w, 1024w, 2560w
   ↓
6. 命名规范
   ├─ product-{name}-{width}w.webp
   ├─ scene-{location}-{width}w.webp
   └─ avatar-{id}-{width}w.webp
   ↓
7. CDN上传和优化
   ├─ 批量上传到CDN
   ├─ 设置缓存规则
   └─ 生成srcset and sizes
   ↓
8. 质量检查
   ├─ 加载速度测试 (WebPageTest)
   ├─ 视觉对比
   └─ 响应式测试 (所有设备)
   ↓
9. 发布和监控
   ├─ 部署到生产环境
   ├─ 监控加载时间和错误率
   └─ 定期优化
```
