# Opula灯具 - 交互与动画设计规范

## 动画总体原则

### 设计哲学
```
品牌调性：高端、优雅、专业、有节奏感

动画关键词：
✓ Smooth（顺滑的缓动）
✓ Meaningful（有意义的提示）
✓ Subtle（细微但有感受）
✓ Performant（不影响性能）

基础参数：
- 持续时间：100ms - 500ms（视复杂度而定）
- 缓动函数：cubic-bezier(0.4, 0, 0.2, 1) 为主
  （Material Design standard easing）
- 帧率目标：60fps (无卡顿)
- GPU加速：transform + opacity 优先
```

### 1.1 页面进入动画 (Page Entry Animation)

#### 首页加载序列
```
时间轴：
0ms    → Hero 标题从左向右淡入
        opacity: 0 → 1
        transform: translateX(-40px) → translateX(0)
        duration: 600ms
        easing: cubic-bezier(0.35, 0, 0.65, 1)

        opacity: 0 → 1
```

#### 页面滚动加载动画
```
当用户滚动到新的section时，元素逐个出现

实现原理：Intersection Observer API

示例：产品卡片列表

.product-card {
  opacity: 0;
  transform: translateY(30px);
  transition: opacity 0.6s ease, transform 0.6s ease;
}

.product-card.in-view {
  opacity: 1;
  transform: translateY(0);
}

JavaScript逻辑：
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.classList.add('in-view');
      // 可选：延迟加载图片
      const img = entry.target.querySelector('img');
      if (img && img.dataset.src) {
        img.src = img.dataset.src;
        img.removeAttribute('data-src');
      }
    }
  });
}, {
  threshold: 0.1,
  rootMargin: '50px'
});

document.querySelectorAll('.product-card').forEach(card => {
  observer.observe(card);
});

动画时间：
- 第1个卡片：延迟 0ms, 持续 600ms
- 第2个卡片：延迟 50ms, 持续 600ms
- 第3个卡片：延迟 100ms, 持续 600ms
- ...（级联效应）

CSS延迟：
.product-card {
  animation: fadeInUp 600ms cubic-bezier(0.4, 0, 0.2, 1) forwards;
}

.product-card:nth-child(1) { animation-delay: 0ms; }
.product-card:nth-child(2) { animation-delay: 50ms; }
.product-card:nth-child(3) { animation-delay: 100ms; }
```

### 2.1 按钮交互 (Button Interactions)

#### 主按钮 (Primary Button)

```
状态转移：
Normal → Hover → Active → Focus

1. Normal State
   background: #D4AF37
   box-shadow: 0 4px 12px rgba(212, 175, 55, 0.3)
   scale: 1
   color: #333333

2. Hover State (鼠标悬停)
   background: #E8D4A8 (亮化)
   box-shadow: 0 8px 20px rgba(212, 175, 55, 0.4) (增强阴影)
   scale: 1.02 (轻微放大)
   color: #333333
   transform: translateY(-2px) (上升效果)
   
   CSS过渡：
   transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1);
   
   时间：300ms（感受用户反应很快）

3. Active/Click State (按下时)
   background: #C4A838 (更暗)
   box-shadow: 0 2px 4px rgba(212, 175, 55, 0.2) (阴影减弱)
   transform: translateY(0px) (恢复正常位置)
   
   时间：100ms - 150ms（立即响应）

4. Focus State (键盘/触屏焦点)
   background: #D4AF37 (保持)
   border: 2px solid #D4AF37
   box-shadow: 0 0 0 4px rgba(212, 175, 55, 0.2)

   outline: none
   
   用于可访问性 (a11y)

  cursor: pointer;
  
  &:hover {
    background: #E8D4A8;
    box-shadow: 0 8px 20px rgba(212, 175, 55, 0.4);

  }
  
  &:active {
    background: #C4A838;
    box-shadow: 0 2px 4px rgba(212, 175, 55, 0.2);
    transform: translateY(0);
  }
  
  &:focus {
    outline: none;
    box-shadow: 0 0 0 4px rgba(212, 175, 55, 0.2);
  }
}
```

## 第三部分：滚动与视差效果 (Scroll Effects)

### 3.1 视差滚动 (Parallax Scroll)

#### Hero 背景视差
```
原理：背景图片以比内容更慢的速度滚动

实现方式（CSS）：
.hero {
  background-image: url('hero-bg.webp');
  background-attachment: fixed; // 固定背景
  background-size: cover;
  background-position: center;
  min-height: 600px;
  
  .hero-content {
    position: relative;
    z-index: 2;
  }
}

兼容性问题：
❌ background-attachment: fixed 在移动端不支持
✓ 使用JS实现移动端版本：

const parallaxElements = document.querySelectorAll('[data-parallax]');

window.addEventListener('scroll', () => {
  parallaxElements.forEach(el => {
    const scrollY = window.scrollY;
    const yPos = scrollY * 0.5; // 0.5 = 50% 速度
    el.style.transform = `translateY(${yPos}px)`;
  });
});

响应式方案：
@media (max-width: 768px) {
  .hero {
    background-attachment: scroll; // 移动端禁用视差
  }
}
```

### 3.2 滚动触发动画 (Scroll Trigger)

#### Sticky 导航栏背景变化
```
当用户向下滚动超过hero区时，导航栏样式改变

实现：JavaScript Scroll Event

window.addEventListener('scroll', () => {
  const scrollY = window.scrollY;
  const navbar = document.querySelector('.navbar');
  
  if (scrollY > 200) { // 200px后触发
    navbar.classList.add('scrolled');
  } else {
    navbar.classList.remove('scrolled');

.navbar.scrolled {

  backdrop-filter: blur(10px); // 毛玻璃效果（可选）
}
```

### 6.2 下拉刷新 (Pull-to-Refresh)

```
原理：用户向下拖动页面时触发刷新

JavaScript：
let startY = 0;
let currentY = 0;
const pullContainer = document.querySelector('.page-content');

pullContainer.addEventListener('touchstart', (e) => {
  startY = e.touches[0].clientY;
});

pullContainer.addEventListener('touchmove', (e) => {
  if (window.scrollY === 0) { // 在页面顶部时
    currentY = e.touches[0].clientY;
    const diff = currentY - startY;
    
    if (diff > 0) {
      const progress = Math.min(diff / 100, 1);
      updatePullRefreshUI(progress);
    }
  }
});

pullContainer.addEventListener('touchend', () => {
  if (currentY - startY > 80) { // 下拉超过80px时刷新
    triggerRefresh();
  }
  resetPullRefresh();
});

function updatePullRefreshUI(progress) {
  const refreshIcon = document.querySelector('.refresh-icon');
  refreshIcon.style.transform = `rotate(${progress * 360}deg)`;
  refreshIcon.style.opacity = progress;
}

function triggerRefresh() {
  // 刷新逻辑...
  showLoadingAnimation();
  setTimeout(() => {
    hideLoadingAnimation();
    resetPullRefresh();
  }, 2000);
}
```

## 第七部分：性能优化建议

### 7.1 动画性能检查清单
```
✓ 使用 transform 和 opacity（GPU加速）
❌ 避免 left/top/width/height 动画（重排）

✓ 使用 will-change 提示浏览器
✓ 使用 requestAnimationFrame（跟随帧率）
✓ 避免过度动画（每个交互1-2个动画）

✓ 测试工具：Chrome DevTools → Performance
✓ 目标帧率：60fps（无卡顿）
✓ 移动端目标：48-60fps（允许轻微波动）
```

### 7.2 关键CSSOptimizations
```css
/* 启用GPU加速 */
.animated {
  will-change: transform, opacity;
  transform: translateZ(0); /* 创建新的栅格层 */
  backface-visibility: hidden; /* 隐藏背面 */
}

/* 批量处理动画 */
.animation-batch {
  animation: batchAnimation 0.3s ease;
}

/* 移动端轻量化 */
@media (max-width: 768px) {
  .parallax {
    background-attachment: scroll; /* 禁用视差 */
  }
  
  .complex-animation {
    animation: none; /* 禁用复杂动画 */
  }
}
```
