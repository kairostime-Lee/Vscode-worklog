# Opula灯具 - 移动端优化方案

## 移动端设计原则

### 核心目标
```
✓ 一指轻松操作 (One-Thumb Operation)
✓ 清晰内容层级 (Clear Information Hierarchy)
✓ 快速加载 (Fast Loading < 3s)
✓ 响应式布局 (Responsive & Fluid)
✓ 触摸友好 (Touch-Friendly)
✓ 完整功能 (Full Feature Parity)
```

### 断点规范（回顾）
```
xs (Extra Small):  320px - 480px  (小手机)
sm (Small):        480px - 768px  (大手机)
md (Medium):       768px - 1024px (平板竖屏)
lg (Large):        1024px - 1280px (平板横屏/笔记本)
xl+ (Desktop):     1280px+       (桌面)

优先级顺序：xs → sm → md → lg → xl
设计方向：移动优先 (Mobile-First)
```

## 第二部分：移动端页面布局

### 2.1 首页移动版布局

#### Hero区移动版
```
高度：300px (xs) / 350px (sm)
内容堆叠：上下排列

结构：
┌─────────────────────┐
│ [产品图]             │ h=160px, aspect-ratio 1/1
│ (圆形或方形)        │
│                    │
│ Illuminate Your    │ h=20px
│ Space...           │
│                    │
│ (副标题)            │
│ ...                │
│                    │
│ [按钮1] [按钮2]     │ w=100% / 2, h=40px
└─────────────────────┘

按钮布局：
- 上按钮：w=100%, h=40px, margin-bottom=8px
- 下按钮：w=100%, h=40px

或者：2列按钮并排（间距8px）

文本对齐：居中（vs 桌面版左对齐）

padding：
- 水平：16px
- 竖直：24px (top/bottom)
```

#### 产品卡片网格 (Mobile)
```
xs/sm: 1列 (100% - 32px padding)
md: 2列

卡片宽度：
- xs: calc(100vw - 32px) / 1
- sm: calc(100vw - 32px) / 1
- md: calc(100vw - 48px) / 2

卡片高度：auto（内容驱动）

间距：
- 行间：16px (xs/sm) / 24px (md+)
- 列间：8px (xs/sm) / 16px (md+)

卡片内容调整：
- 图片：100% 宽度，保持1:1比例
- 标题：2行max, 14px (sm) / 16px (md+)
- 价格：16px Bold
- 按钮：w=100%, h=40px
```

### 2.2 产品详情页 - 移动版

```
xs/sm: 1列（全宽）
md+: 2列

布局序列：
1. 图片库（全宽，高度自适应）
2. 产品信息（全宽，sticky顶部）
3. 完整描述（全宽）
4. 规格表（横向滚动，如果表格过宽）
5. 评价（全宽）
6. 相关产品（全宽，1列）

产品信息区 (Mobile) - Sticky设计：

固定高度区域，向上滚动时sticky顶部：
┌──────────────────┐

│ ⭐ 评分          │
│ 价格             │

当向下滚动时：
- 图片区：向上滚动消失
- 信息区：sticky在顶部（导航栏下方）
- 用户随时可点击"Add to Cart"
```

### 5.3 滑动与拖拽

```

<div class="swiper-container">

const swiper = new Swiper('.swiper-container', {

实现方案2：CSS Scroll Snap

}

.gallery-img {

实现方案3：原生Touch Events
const gallery = document.querySelector('.image-gallery');
let startX = 0;
let scrollLeft = 0;

gallery.addEventListener('touchstart', (e) => {
  startX = e.touches[0].clientX;
  scrollLeft = gallery.scrollLeft;
});

gallery.addEventListener('touchmove', (e) => {
  const walk = (e.touches[0].clientX - startX) * 2; // 倍数越大越灵敏
  gallery.scrollLeft = scrollLeft - walk;
});
```

## 第六部分：移动端性能优化

### 6.1 图片加载策略

```
首屏图片：
- 高优先级加载
- 使用WebP + JPG fallback
- 限制尺寸：xs=480w, sm=768w

非首屏图片：
- 懒加载 (loading="lazy")
- Intersection Observer
- 延迟加载直到用户滚动到

图片尺寸优化（移动）：
┌──────────────┬───────┬────────┐
│ 设备         │ 宽度  │ 文件大小│

│ sm 手机      │ 768px │ ~100KB │
│ md 平板      │1024px │ ~180KB │
└──────────────┴───────┴────────┘

Picture标签示例：
```

## 第七部分：移动端安全与无障碍

### 7.1 可访问性 (Accessibility)

```
WCAG 2.1 AA 标准遵循：

颜色对比度：
- 文本与背景：最少 4.5:1 (常规文本)
- UI 组件：最少 3:1 (大型文本或icon)

键盘导航：
- Tab 顺序合理
- 所有交互可通过键盘访问
- Focus 可见（outline或border）

屏幕阅读器：
- 语义HTML (<button>, <nav>, <main>)
- ARIA labels 补充说明
- Alt text for images
- Form labels associated with inputs

移动无障碍：
- 按钮最小 44×44px
- 文字最小 14px
- 颜色不是唯一区分方式
- 动画可禁用

示例代码：
<!-- 优秀 -->
<button aria-label="Add to cart">
  <svg aria-hidden="true">...</svg>
</button>

<!-- 避免 -->
<div class="btn" onclick="addCart()">Add</div>

<!-- 图片alt文本 -->
<img src="lamp.webp" alt="Luna Pendant Light - Nordic Design">

<!-- Form关联 -->
<label for="email">Email</label>
<input id="email" type="email">
```

```
API安全：

存储安全：

```
